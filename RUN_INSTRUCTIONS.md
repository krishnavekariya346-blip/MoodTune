# How to Run MoodTune App

## Option 1: Using Android Studio (Recommended)

1. **Open Android Studio**
2. **Open Project**: File → Open → Select the `MT` folder
3. **Wait for Gradle Sync**: Android Studio will automatically download dependencies and sync the project
4. **Connect Emulator or Device**:
   - For Emulator: Tools → Device Manager → Create/Start an emulator
   - For Real Device: Enable USB Debugging and connect via USB
5. **Run the App**: Click the green "Run" button or press `Shift+F10`

## Option 2: Using Command Line

### Prerequisites:
- Java JDK 11 or higher installed
- Android SDK installed
- Gradle wrapper files (will be auto-generated by Android Studio)

### Steps:
1. Navigate to the MT folder:
   ```powershell
   cd MT
   ```

2. Build the APK:
   ```powershell
   .\gradlew.bat assembleDebug
   ```

3. Install on connected device/emulator:
   ```powershell
   .\gradlew.bat installDebug
   ```

   OR use ADB directly:
   ```powershell
   adb install app\build\outputs\apk\debug\app-debug.apk
   ```

## Important Notes:

1. **Backend Servers**: Make sure your backend servers are running:
   - Face Detection: `http://192.168.1.9:5000/analyze` (or `http://10.0.2.2:5000/analyze` for emulator)
   - Chatbot: `http://192.168.1.9:5000/predict` (or `http://10.0.2.2:5000/predict` for emulator)

2. **Firebase Setup**: The `google-services.json` file is already in place, but make sure:
   - Firebase project is configured correctly
   - Authentication is enabled
   - Firestore database is set up

3. **First Run**: 
   - The app will start with the Login screen
   - Sign in with Google
   - You'll be taken to the Home Page

## Troubleshooting:

- **Gradle Sync Failed**: Make sure you have internet connection and Android SDK is properly configured
- **Build Errors**: Check that all dependencies in `build.gradle.kts` are correct
- **Backend Connection Issues**: Verify your backend servers are running and accessible
- **Firebase Errors**: Check that `google-services.json` is in the `app/` directory
